Geo — Daily Geography Outline Quiz

PRD · 2026-07-11 · synthesized from in-session

A daily geography game: guess the country by letter, Hangman-style, against a countdown clock, while its outline — and its neighbors' — draw themselves in.

Problem

Players who enjoy daily puzzle games (Wordle, Globle, Worldle, Flagle) want a geography-flavored entry in that genre, but a single flash-card 'guess the country from its outline' round is over in ~10 seconds and doesn't sustain the daily-ritual habit those games rely on. None of the existing games combine a drawing reveal with a neighbor-cascade hint structure under real time pressure — that combination is the differentiator this product is betting on.

Solution

A single daily target country, identical for every player. The player guesses individual letters (literal Hangman) to fill in the target country's name, racing a flat 60-second countdown clock. Each wrong letter guess costs time, scaled inversely with the target's unique letter count (fewer unique letters = harsher penalty, since the puzzle is otherwise easier). The clock drives everything: the target's outline draws alone for the first ~40-50% of the round, then three neighboring countries' outlines begin appearing too, positioned at their real-world compass direction relative to the target, with name labels revealing letters in randomized order automatically. Round ends on solve, give-up, or timeout. A streak counter (localStorage for now) tracks consecutive days solved, and a spoiler-safe shareable result (day number, time remaining or failed, colored guess-pattern row) lets players compare outcomes without revealing the answer.

UX flow

1. Round start: clock at 0:60, target outline starts drawing, trivia fact overlaid, Hangman blanks shown empty. 2. Guessing: player guesses letters one at a time; correct fills all instances; wrong subtracts time per the unique-letter-count penalty tier. 3. Neighbor reveal (staggered onset): once time remaining drops below ~40-50%, three neighbor outlines draw in at compass-accurate positions with labels revealing letters in randomized order automatically. 4. Round end: solved (full name guessed), given up (button), or timeout (clock hits 0) — all three are the only exits. 5. Result & share: solved shows day number, time remaining, guess-pattern row, and flag/country name (safe since solved); failed shows the same minus country identity, which stays hidden.

Technical architecture

Scaffold: Vite + React + TypeScript, fully static app for now (no backend at all until Supabase lands later). Frontend: React + Framer Motion, pathLength/pathSpacing draw-on for target + 3 neighbor slots, Hangman input UI (keyboard mode TBD) + small numeric countdown top-right. Geo data: world-atlas TopoJSON at 50m resolution (10m stutters, 110m too coarse), equal-area/equirectangular projection (not Mercator). Neighbor data: restcountries.com borders field, compass-direction slot assignment from centroids, deterministic subset for >3 neighbors seeded like the daily country, islands (0 neighbors) included as harder days. Daily puzzle: deterministic client-side hash — hash(UTC date) mod country_count picks today's target and neighbor subset, no backend call, guarantees every player sees the same country without a server, works for any future date automatically. Game state: client-side state machine for clock/penalties/guesses/hint-thresholds. Persistence: localStorage for streak state for now — Supabase explicitly deferred, and even when added later has no numeric leaderboard, only cross-device streak sync. Trivia: LLM-generated for all ~190 countries with a one-time human review pass. Rejected: Leaflet/MapLibre GL (wrong tool for fixed-viewport animated outlines); GSAP+DrawSVG viable fallback if Framer Motion's pathLength is insufficient.

Data model

Country { iso_code, name, unique_letters (drives penalty tier), outline_path, centroid, neighbor_codes[], trivia_fact, is_island }. DailyPuzzle { date (UTC PK), target_iso_code, neighbor_iso_codes[] (deterministic subset, seeded by date), neighbor_slots[] (iso_code + resolved compass_direction) }. PenaltyTier (config) { min_unique_letters, max_unique_letters, penalty_seconds — illustrative: (0,5,20) (6,9,15) (10,null,10) }. PlayerRound (localStorage) { puzzle_date, outcome (solved/failed/gave_up), time_remaining, share_string }. StreakState (localStorage, Supabase later for cross-device sync only) { current_streak, longest_streak, last_played_date }.

Pricing

Free, no monetization, for v1 — a deliberate constraint carried through the entire stack (free/open geo data, free animation tooling, no paid backend tier required while using localStorage). Revisit only if the game finds real traction.

Roadmap

v0 (prototype): single hard-coded country, draw-on outline animation working end to end with real 50m TopoJSON data, no Hangman/clock/scoring yet — validates animation feel and projection choice. v1 (MVP): full loop — Hangman letter guessing, 60s clock with unique-letter-scaled penalties, staggered target-then-neighbor hint reveal with compass-accurate positioning, give-up action, localStorage-backed streak, spoiler-safe share string, LLM-generated+reviewed trivia for all countries, disputed-territory handling left at data-source default. Later: Supabase-backed cross-device streak sync (still no numeric leaderboard), playtested tuning of the 60s budget and penalty tiers, refined compass-anchor collision handling, possible untimed/practice mode for accessibility.

Risks

60-second flat clock may be miscalibrated for very long or very short country names even with unique-letter-scaled penalties — needs aggressive playtesting across name-length extremes. Coastline complexity can stutter the draw animation for detailed-border countries, now applying to up to 4 simultaneous outlines instead of 1. Compass-slot collisions are unresolved in detail — multiple neighbors can round to the same direction anchor, no resolution rule designed yet. Deliberation time is now part of the score, not just guess accuracy — an accepted tradeoff worth monitoring in early feedback. Inheriting the geo-data source's disputed-territory defaults rather than an explicit policy could surprise players in edge cases, accepted as low-priority for a hobby project. LLM-generated trivia needs a real review pass to avoid shipping ungrounded or subtly wrong facts.

Open questions

Exact time-penalty values per unique-letter tier (illustrative only, needs playtesting). Exact compass-anchor scheme (e.g. 8-point compass) and collision-resolution rule for neighbors sharing a direction. Exact deterministic-subset selection algorithm for countries with more than 3 neighbors (seeded by date, but the selection rule itself unspecified). Exact visual format of the share string's guess-pattern row. Trivia fact review process specifics — who reviews, what bar 'verifiably true' needs to clear. Timing/trigger for migrating streak persistence from localStorage to Supabase (deferred indefinitely). On-screen keyboard vs. physical keydown capture for letter input, significant for mobile UX.
